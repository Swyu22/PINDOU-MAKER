# 新模型接手 Prompt（完整版，可直接复制发送）

你现在接手一个已有项目，请严格按以下流程工作（不要直接开始写代码）：

## 目标
你的目标不是单次回答聪明，而是保证项目在跨设备/跨模型协作中连续迭代不失真。
文件系统是唯一可靠状态源（SSOT），不得仅依赖会话记忆。

## 第一步：读取与对齐（必须先做）
请先按顺序读取并总结：
1. `CLAUDE.md`
2. `/.cloud.md`
3. 本次目标模块的 `README.md`（或 `_module.md`，如果存在）
4. 我指定的目标文件与相邻关键文件（如果我已指定）
5. 如果任务涉及架构/跨模块/接口变更，请继续读取：
   - `/docs/00-context/PROJECT_MAP.md`
   - `/docs/10-spec/*`（相关文档）
   - `/docs/20-plan/current-iteration.md`
   - `/docs/30-decisions/*`（相关 ADR）

## 第二步：输出你的理解（不超过 8 条）
请输出：
- 当前目标（你理解的）
- 本次范围（会改哪些文件 / 不改哪些）
- 关键约束（组件库/架构/质量红线）
- 验收标准
- 风险/不确定点
- 你还需要读取哪些文件（如 spec/ADR/plan）
- 本次预计需要回写哪些文档（至少包含 `.cloud.md` 与 `ai/sessions`）

## 第三步：给出执行计划（先计划再执行）
请列出：
1. 执行步骤（按顺序）
2. 验证方法（测试 / lint / build / hook）
3. 回写清单（代码文件 + 文档文件）
4. 是否需要新增 ADR（若存在架构/接口关键决策）

## 第四步：执行时必须遵守的规则
- 渐进式上下文加载：按任务复杂度决定读取范围，禁止默认全量灌入所有文档
- Hook / 自动检查 > Prompt 口头约束，Hook 失败不得强行推进
- 改完代码必须回写文档：
  1) 关键文件头说明（若职责/依赖/输出变化）
  2) 模块 README（若结构/职责变化）
  3) `PROJECT_MAP.md`（若架构/模块边界变化）
  4) `/.cloud.md`
  5) `docs/20-plan/current-iteration.md`（更新 checklist）
  6) `docs/30-decisions/*`（如有关键决策）
  7) `ai/sessions/YYYY-MM-DD.md`
- 质量红线（默认）：单文件≤800行、单函数≤30行、嵌套≤3层、分支≤3个（如项目文档另有规定，以项目文档为准）

## 第五步：工作完成后必须输出并建议写入的文件内容
### A. 日常任务默认输出（Daily）
```md
## Session Summary (Daily)

### Last Done
- 已完成：
- 已验证：

### Next Actions（Priority）
1. P0:
2. P1:
3. P2:

### Key Decisions
- 本次无新增关键决策
# 或
- 决策：
- 原因：
- 影响：
```
建议写入：`/ai/sessions/YYYY-MM-DD.md`

### B. 复杂任务/里程碑任务输出（Milestone）
```md
### 今日目标完成情况（对照最初目标逐条勾选）
- [ ] 目标1：
- [ ] 目标2：

### 关键变更清单（文件路径 + 做了什么 + 为什么）
- `path/to/file`：做了什么；为什么

### 关键决策（ADR）（背景/方案A/方案B/选择/影响）
- 背景：
- 方案A：
- 方案B：
- 选择：
- 影响：

### 未完成 & 下一步（按优先级）
1. P0：
2. P1：
3. P2：

### 可复用提示词（可选）
1. ...
2. ...
```
建议写入：
- `/ai/sessions/YYYY-MM-DD.md`
- `/docs/30-decisions/adr-XXXX-<slug>.md`（如有关键决策）

## 第六步：输出格式要求
在你开始执行前，请先输出：
1) 你读取了哪些文件
2) 你的理解摘要（≤8条）
3) 执行计划
4) 回写清单

在你完成执行后，请输出：
1) 已完成内容摘要
2) 验证结果
3) Session Summary（Daily 或 Milestone）
4) 建议写入路径列表
